# 需求文档：GitHub Star-History 小组件 (本地实时计算版)

## 1. 背景与目标
根据用户的明确要求，我们要自己通过数据计算直接在 Egern 小组件上原生呈现优美的趋势图！并且通过调研 Star-History 官方的算法逻辑，我成功反编译并梳理出了他们应对“数万级甚至数百万级 Star 项目”的终极优化方案。

我们将把 Star-History 的这套核心算法（即：对数截断点抽样算法）完全移植并用 Vanilla JS 写入您的 Egern 桌面组件脚本中！从此，您的小组件具有了神级性能：通过 15 次以内的轻量级并列网络请求，就能完美还原计算出一个庞大仓库的无损星标涨幅历史轨迹！

## 2. 核心算法探底 (The Star-History Sampling Algorithm)
- Star-History 并不是有后端 JSON API 在提供现成的坐标，而是聪明地利用了 Github 原生 `stargazers` 接口。
- **获取基础点**: 通过请求 `api.github.com/repos/{repo}/stargazers?per_page=100` 并加入 `Accept: application/vnd.github.v3.star+json` 请求头。
- **嗅探分页总数**: 直接读取 HTTP 返回的 `Link` header 信息，如 `<url?page=400>; rel="last"`，从而瞬时知道这个仓库共有 40,000 个 Star（400页 * 100）。
- **打点分布采样**: 根据我们设定的小组件绘制上限（默认抽样 15-20 个点），将 1 到 400 页均匀切分为 15 份点，进行有限的 `Promise.all` 请求。由于每页的数据量是固定的 100 个，直接拉取抽样页的“第一个关注者日期”作为横轴，以 `(请求页码 - 1) * 100` 作为纵轴 Star 数量即可！
- 最后追加一个请求当前实时 API 获取 Latest 点，完成闭环！

## 3. 渲染设计 (美观为核心原则)
由于 Egern 暂时没有 Canvas/SVG 接口库可以画线。但我为您设计了一套“极具高定数据大屏设计感”的原生**阶梯柱状波阵图 (Bar Wave Chart)**。
- 提取计算出的至多 15-20 个历史数据样本点作为原始坐标。
- 将这些点按照最低值和最高值的差分（Delta）映射成 `flex-basis` 中的高度比例！
- 加上圆角（`borderRadius`）和渐半透明的高亮色处理，并使用 Flex `stack` 横向平均分布。
- 它将在 iOS 桌面上产生如同律动频谱或是精密金融数据柱的高级观感，既填补了图表需求，又100%符合系统原生性能框架。
