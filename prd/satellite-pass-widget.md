# 需求文档：卫星过境提醒小组件

## 1. 背景与目标
面向天文爱好者与摄影用户，提供下一次可见卫星（如 ISS）过境的关键信息与可视化提示，帮助用户在最佳时间段观察。

**核心目标：**
- 获取下一次可见过境时间与方位信息
- 适配多尺寸小组件（主屏幕 + 锁屏）
- 提供清晰的观测提示（方位、最大仰角、持续时长）
- 具备缓存与异常降级能力

## 2. 用户画像与使用场景
- 天文爱好者：想快速了解下一次过境时间与方位
- 摄影用户：提前准备设备与构图
- 科普兴趣者：希望日常获得天文事件提示

## 3. 数据源与公开 API
**推荐 API：** N2YO 可见过境接口（需 API Key）
- 过境接口示例（ISS）：
  - `https://api.n2yo.com/rest/v1/satellite/visualpasses/{satid}/{lat}/{lon}/{alt}/{days}/{min_visibility}?apiKey={key}`
- 关键字段：
  - `passes[].startUTC`：开始时间（UTC 秒）
  - `passes[].endUTC`：结束时间（UTC 秒）
  - `passes[].maxEl`：最大仰角（度）
  - `passes[].startAzCompass`：起始方位（如 WNW）
  - `passes[].endAzCompass`：结束方位
  - `passes[].duration`：持续时长（秒）

**容错补充：**
- 无可见过境：显示“未来 X 天无可见过境”
- API Key 缺失：展示配置提示
- 经纬度缺失：引导用户配置经纬度

## 4. 功能需求
### 4.1 核心功能
- 获取**下一次可见过境**记录（取第一条）
- 计算倒计时（天/小时/分钟）
- 展示方位、最大仰角、持续时长
- 展示“数据状态”（实时/缓存）

### 4.2 缓存与刷新策略
- 默认缓存 30 分钟
- 网络失败：优先回退缓存
- 无缓存且请求失败：显示错误卡

### 4.3 交互行为
- 点击小组件打开过境详情页面（N2YO 或自定义链接）

## 5. UI/UX 设计规范
### 5.1 视觉基调
- 深空蓝背景 + 绿色/蓝色霓虹点缀
- 方位信息使用指南针/罗盘图标

### 5.2 布局适配
#### systemSmall
- 顶部：标题 + 卫星图标
- 中部：倒计时大字
- 底部：最大仰角与方位

#### systemMedium
- 左侧：倒计时 + 状态标签
- 右侧：方位、最大仰角、持续时长三行信息

#### systemLarge
- 顶部：标题 + 状态
- 中部：倒计时 + 方位箭头
- 底部：详细信息（起止方位、最大仰角、持续时长）

#### accessoryCircular
- 中心：倒计时小时/分钟
- 辅助：小卫星图标

#### accessoryRectangular
- 单行标题 + 倒计时 + 方位

#### accessoryInline
- 单行：`🛰️ T-2h 15m 最大仰角 63°`

## 6. 组件参数（env）
- `TITLE`：标题（默认“卫星过境提醒”）
- `SATELLITE_ID`：卫星 ID（默认 ISS: 25544）
- `LAT`/`LON`/`ALT`：位置经纬度与海拔
- `DAYS`：查询天数（默认 2）
- `MIN_VISIBILITY`：最小可见时长（默认 60 秒）
- `API_KEY`：N2YO API Key
- `REFRESH_MINUTES`：刷新间隔（默认 30）
- `FORCE_REFRESH`：强制刷新

## 7. 状态与异常处理
- **正常：** 展示倒计时 + 过境信息
- **无可见过境：** 显示“未来 X 天无可见过境”
- **配置缺失：** 显示缺失字段提示
- **请求失败：** 使用缓存；无缓存则显示错误

## 8. 关键指标
- 首屏渲染成功率 > 99%
- 请求平均耗时 < 2s
- UI 元素不重叠，所有尺寸可读性达标

## 9. 开发计划（里程碑）
1. **需求确认与字段梳理**：明确 API 字段映射与展示优先级
2. **数据层实现**：请求/解析/缓存/异常降级
3. **倒计时与方位逻辑**：计算倒计时、方位与时长格式化
4. **多尺寸 UI 组件**：Small/Medium/Large/锁屏布局
5. **交互与配置**：点击跳转、env 参数校验
6. **测试与优化**：空数据、错误、极端时间边界
