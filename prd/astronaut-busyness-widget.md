# 需求文档：宇航员现在比你还忙吗小组件

## 1. 背景与目标
面向喜欢航天、地理与轻娱乐表达的用户，提供一个“把实时太空状态翻译成一句有趣判断”的小组件。它不是单纯展示 ISS 位置，而是回答一个更容易传播的问题：**此刻宇航员到底有没有比你更忙**。

**核心目标：**
- 基于公开 API 获取 ISS 实时位置与当前在太空中的人数
- 结合用户本地时间、与 ISS 的地理关系、昼夜状态，生成“忙碌指数”
- 输出一句强结论文案，例如“宇航员现在比你忙 42%”
- 适配多尺寸小组件，并在无网络时优先展示最近一次缓存结果

## 2. 用户画像与使用场景
- 航天爱好者：想实时知道 ISS 在哪里、太空里现在有多少人
- 主屏装饰用户：希望组件有一点数据感，但核心是好玩而不是工具味
- 内容分享用户：想把“宇航员比我还忙吗”这种结论截图分享

典型场景：
- 早上看一眼：今天开始工作前，先看“太空里的人有没有更惨”
- 晚上刷主屏：知道 ISS 当前飞到哪里，顺便看一句娱乐化文案
- 锁屏 glance：只看结论和一个数字，不需要展开细节

## 3. 数据源与公开 API
### 3.1 主数据源
**Open Notify**
- ISS 当前位置：
  - `http://api.open-notify.org/iss-now.json`
- 当前在太空中的人数：
  - `http://api.open-notify.org/astros.json`

### 3.2 数据字段
`iss-now.json`
- `timestamp`：服务端时间戳
- `iss_position.latitude`
- `iss_position.longitude`

`astros.json`
- `number`：当前太空总人数
- `people[].name`
- `people[].craft`

### 3.3 本地计算与补充逻辑
- 用户当前位置：通过 `env.LAT` / `env.LON`
- 本地昼夜：本地时间或可选接入 `sunrise-sunset.org`，MVP 阶段允许仅基于小时段判断
- 地理距离：本地计算用户与 ISS 地面投影点的球面距离
- 忙碌指数：由以下因素组成
  - 太空人数
  - ISS 与用户的距离
  - 本地时间段
  - ISS 是否接近用户头顶区域

### 3.4 容错策略
- ISS 接口失败但人数接口成功：仍展示“太空中有 X 人”与降级文案
- 人数接口失败但 ISS 接口成功：展示轨迹信息与“忙碌指数待定”
- 两者都失败：优先使用缓存；无缓存则显示错误卡

## 4. 产品定义
### 4.1 一句话卖点
把太空实时数据翻译成一句轻松、可传播的判断：**“宇航员现在比你还忙吗？”**

### 4.2 核心输出
- 忙碌指数：0-100
- 结论文案：`比你更忙` / `差不多忙` / `你今天更惨`
- 太空人数：当前在太空中的总人数
- ISS 位置：经纬度或区域描述
- 与用户距离：公里
- 彩蛋文案：基于指数与时间段生成

### 4.3 忙碌指数建议算法
初版只追求稳定和可解释，不做“真忙碌程度”的伪科学建模。

基础分 50，叠加项如下：
- 太空人数加分：
  - `number <= 3`：+0
  - `4-7`：+8
  - `8-12`：+15
  - `>12`：+20
- ISS 与用户距离：
  - `< 1000km`：+20
  - `1000-5000km`：+10
  - `>5000km`：+0
- 时间段修正：
  - 本地 09:00-18:00：+8
  - 本地 00:00-06:00：-10
- 可见感修正：
  - 若 ISS 在用户纬度附近且距离较近：+8

最终限制在 `0-100`。

对应结论：
- `0-34`：`你今天更忙`
- `35-64`：`你们差不多忙`
- `65-100`：`宇航员现在更忙`

### 4.4 文案风格
- 主文案要短、像标题
- 副文案偏轻松但不油腻
- 示例：
  - `宇航员现在更忙`
  - `他们正在你头顶附近打工`
  - `太空中 11 人，ISS 距你 1820km`

## 5. 功能需求
### 5.1 核心功能
- 拉取 ISS 实时位置
- 拉取当前在太空中的人数
- 根据用户位置计算与 ISS 的地面距离
- 生成忙碌指数与结论文案
- 根据组件尺寸输出不同信息密度

### 5.2 可选增强功能
- 显示太空人员名单 Top 3
- 显示 ISS 所在半球或海域推断
- 增加“过去 90 分钟绕地球一圈”的动态提示文案
- 组件点击后跳转到航天相关页面

### 5.3 缓存与刷新策略
- 默认刷新：`5` 分钟
- 缓存可用期：`10` 分钟
- 当网络失败时：
  - 先回退缓存
  - 缓存超过 24 小时时标记为“旧数据”
- 支持 `FORCE_REFRESH`

## 6. UI/UX 设计规范
### 6.1 视觉基调
- 风格：太空雷达屏 + 轻信息图
- 主色：电蓝、荧光绿、深空黑
- 视觉中心：一个强结论 + 一个忙碌指数数字

### 6.2 布局适配
#### systemSmall
- 顶部：标题 + ISS 图标
- 中部：结论文案
- 底部：忙碌指数 + 太空人数

示例：
- `宇航员更忙`
- `指数 78`
- `太空中 11 人`

#### systemMedium
- 左侧：大号结论与指数
- 右侧：ISS 距离、太空人数、当前位置摘要
- 底部：live/cached 状态

#### systemLarge
- 顶部：标题、状态
- 中部：结论、指数、彩蛋文案
- 底部：更多细节
  - ISS 经纬度
  - 与用户距离
  - 太空人数
  - 人员名单摘要

#### accessoryCircular
- 中心：指数数字
- 辅助：小字 `忙/闲`

#### accessoryRectangular
- 单行结论 + 指数
- 次行：`11人 | 1820km`

#### accessoryInline
- 单行：`ISS 现在比你忙 78 分`

## 7. 组件参数（env）
- `TITLE`：标题，默认 `宇航员现在比你还忙吗`
- `LAT`：用户纬度，必填
- `LON`：用户经度，必填
- `ACCENT_COLOR`：强调色，默认 `#38BDF8`
- `REFRESH_MINUTES`：刷新间隔，默认 `5`
- `FORCE_REFRESH`：是否强制刷新
- `OPEN_URL`：点击组件跳转链接，可选
- `DISTANCE_UNIT`：`km` / `mi`，默认 `km`

## 8. 状态与异常处理
- **正常：**
  - 展示结论文案、指数、人数、距离
- **部分降级：**
  - 人数缺失：显示 `太空人数待定`
  - ISS 位置缺失：显示 `轨道定位暂不可用`
- **配置缺失：**
  - 未填写 `LAT/LON`，展示配置提示
- **请求失败：**
  - 使用缓存
  - 无缓存则显示错误卡：`暂时联系不上太空`

## 10. 验收标准
- 配置完整时，小组件可以稳定渲染，不因单接口失败直接崩溃
- 中小尺寸首屏必须一眼看到“结论 + 指数”
- 忙碌指数规则可解释，不出现明显随机跳变
- 网络失败时可以优先展示缓存
- accessory 系列在一行内保持可读

## 11. 里程碑
1. 接入 Open Notify 两个接口并完成字段解析

3. 完成文案生成与缓存逻辑
4. 完成多尺寸布局
