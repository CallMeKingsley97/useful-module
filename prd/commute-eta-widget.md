# 需求文档：通勤时间预测小组件

## 1. 背景与目标
面向通勤用户，提供家↔公司两条固定路线的实时预估到达时间（ETA），帮助快速判断通勤成本与出发时机。

**核心目标：**
- 展示当前 ETA 与拥堵等级
- 支持双路线切换或并列展示
- 适配多尺寸小组件
- 具备缓存与异常降级能力

## 2. 用户画像与使用场景
- 上班族：出门前快速评估通勤耗时
- 司机/骑行用户：判断是否需要改路线或延迟出发
- 城市通勤者：习惯固定路线

## 3. 数据源与公开 API
**推荐 API：** 高德地图 Web 服务 - 路径规划接口
- 驾车路径规划：
  - `https://restapi.amap.com/v3/direction/driving?origin=lng,lat&destination=lng,lat&key=KEY&extensions=base&strategy=0`
- 关键字段：
  - `route.paths[0].duration`：预计耗时（秒）
  - `route.paths[0].distance`：距离（米）
  - `route.paths[0].traffic_lights`：红绿灯数量（可选）

**容错补充：**
- API Key 缺失：提示配置
- 失败时回退缓存

## 4. 功能需求
### 4.1 核心功能
- 获取家→公司、公司→家两个方向 ETA
- 计算并展示时长（分钟/小时）
- 展示距离与拥堵等级（基于速度/时长阈值）

### 4.2 缓存与刷新策略
- 默认缓存 10 分钟
- 网络失败：优先回退缓存
- 无缓存且请求失败：显示错误卡

### 4.3 交互行为
- 点击小组件打开高德/系统地图（可配置 URL Scheme）

## 5. UI/UX 设计规范
### 5.1 视觉基调
- 清爽通勤风格：浅灰/深蓝主色
- 绿色/橙色/红色表示拥堵等级

### 5.2 布局适配
#### systemSmall
- 顶部：标题 + 车辆图标
- 中部：当前方向 ETA
- 底部：距离/拥堵颜色提示

#### systemMedium
- 左侧：家→公司 ETA
- 右侧：公司→家 ETA
- 底部：刷新状态

#### systemLarge
- 顶部：标题 + 状态
- 中部：双向 ETA + 距离
- 底部：拥堵说明（快/一般/拥堵）

#### accessoryCircular
- 中心：单向 ETA
- 辅助：小车图标

#### accessoryRectangular
- 单行：家→公司 ETA + 拥堵色

#### accessoryInline
- 单行：`🚗 去公司 35 分钟`

## 6. 组件参数（env）
- `TITLE`：标题（默认“通勤时间预测”）
- `HOME`：家坐标（`lng,lat`）
- `WORK`：公司坐标（`lng,lat`）
- `API_KEY`：高德 API Key
- `STRATEGY`：路径策略（默认 0）
- `REFRESH_MINUTES`：刷新间隔（默认 10）
- `FORCE_REFRESH`：强制刷新

## 7. 状态与异常处理
- **正常：** 展示双向 ETA 与拥堵等级
- **配置缺失：** 显示缺失字段提示
- **请求失败：** 使用缓存；无缓存则显示错误

## 8. 关键指标
- 首屏渲染成功率 > 99%
- 请求平均耗时 < 2s
- UI 元素不重叠，所有尺寸可读性达标

## 9. 开发计划（里程碑）
1. **需求确认与字段梳理**：API 字段映射与阈值定义
2. **数据层实现**：请求/解析/缓存/异常降级
3. **通勤逻辑**：ETA 格式化、拥堵等级判定
4. **多尺寸 UI 组件**：Small/Medium/Large/锁屏布局
5. **交互与配置**：点击跳转、env 参数校验
6. **测试与优化**：空数据、错误、边界情况
