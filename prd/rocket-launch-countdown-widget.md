# 需求文档：极简宇航发射倒计时小组件

## 1. 背景与目标
面向航天爱好者与科技从业者，提供“下一次火箭/航天器发射”的**极简倒计时**与**关键信息卡**，在不依赖系统能力的前提下，仅使用公开 API 拉取数据并通过 Egern 小组件 DSL 渲染。

**核心目标：**
- 获取最近一次发射任务信息并展示倒计时
- 兼容多尺寸小组件（主屏幕 + 锁屏）
- UI 质感精美、信息密度适中、读秒清晰
- 具备缓存与异常降级能力

## 2. 用户画像与使用场景
- 航天/科技爱好者：每天想快速知道下一次发射的时间与载荷信息
- 科技从业者：以“极简科技感”小组件装饰主屏幕
- 资讯关注者：希望无需打开 App 即可了解下一次发射倒计时

## 3. 数据源与公开 API
**推荐 API：** Launch Library 2（The Space Devs）
- 基础接口：`https://ll.thespacedevs.com/2.2.0/launch/upcoming/`
- 过滤条件：`?limit=1&mode=detailed` 或按窗口时间过滤
- 字段示例：
  - `name`：任务名称
  - `net`：发射时间（UTC）
  - `status.name`：状态（Go/To Be Determined/Partial）
  - `pad.name`：发射场
  - `location.name`：地点
  - `rocket.configuration.name`：火箭型号
  - `mission.description`：任务简介
  - `image`：任务图片

**容错补充：**
- 若 `image` 为空，使用预置图标或渐变背景
- 若 `net` 为 TBD，显示“时间待定”并隐藏倒计时

## 4. 功能需求
### 4.1 核心功能
- 获取**下一次发射**任务信息
- 计算倒计时（天/小时/分钟）
- 显示关键字段：任务名、火箭型号、发射场、状态
- 显示“当前数据状态”（实时/缓存）

### 4.2 缓存与刷新策略
- 默认缓存 10 分钟（参考 [`modules/github-trending.js`](../modules/github-trending.js:1) 的缓存策略）
- 网络失败：优先回退缓存
- 无缓存且请求失败：显示错误卡
- 支持通过 `env` 配置刷新间隔与强制刷新

### 4.3 交互行为
- 点击小组件打开任务详情页（`url` 指向任务官网或 Launch Library 页面）
- 锁屏组件仅显示关键倒计时信息

## 5. UI/UX 设计规范（精美重点）
### 5.1 视觉基调
- **风格：** 极简科幻感 + 深色系
- **色彩：** 深空蓝/暗紫 + 霓虹点缀（如青蓝、橙红）
- **字体：** 大数字突出倒计时，细字展示补充信息
- **结构：** 强层级、留白充分、信息块卡片化

### 5.2 关键视觉元素
- **倒计时主数字**：
  - 大号字体（32~40）
  - 使用渐变高亮或微光阴影
- **火箭图标**：
  - SF Symbol：`sf-symbol:rocket.fill`
  - 小尺寸用于标题区，视觉点缀
- **状态标签**：
  - `GO` / `TBD` / `HOLD` 等状态色块
  - 绿色/黄色/灰色区分
- **背景**：
  - 渐变：从深蓝到暗紫
  - 可选星点噪点（后期可扩展 Base64 图）

### 5.3 布局适配
#### systemSmall（主屏幕小号）
- 顶部：火箭图标 + “Next Launch”
- 中部：倒计时主数字（`HH:MM` 或 `Xd Hh`）
- 底部：任务简称（单行截断）

#### systemMedium（主屏幕中号）
- 左侧：倒计时大数字 + 状态标签
- 右侧：任务名、火箭型号、发射场（3 行）
- 底部：刷新状态（live/cached）

#### systemLarge（主屏幕大号）
- 顶部：任务名 + 状态标签
- 中部：大倒计时 + 火箭型号 + 发射场
- 底部：任务摘要（2~3 行）

#### accessoryCircular（锁屏圆形）
- 中心：倒计时小时数（`H`）或分钟数（`M`）
- 外圈：细线/渐变环（模拟）

#### accessoryRectangular（锁屏矩形）
- 左侧：火箭图标 + 倒计时
- 右侧：任务简称 + 状态

#### accessoryInline（锁屏内联）
- 单行：`🚀 T-3h 12m Falcon 9`

## 6. 组件参数（env）
- `TITLE`：标题（默认 “Next Launch”）
- `ACCENT_COLOR`：强调色（默认 `#22D3EE`）
- `REFRESH_MINUTES`：刷新间隔（默认 10）
- `FORCE_REFRESH`：强制刷新（默认 false）
- `API_URL`：可覆盖默认 API

## 7. 状态与异常处理
- **正常：** 展示倒计时 + 任务信息
- **TBD：** 显示“时间待定”，倒计时隐藏
- **请求失败：** 使用缓存；无缓存则显示“暂不可用”
- **字段缺失：** 兜底文本“Unknown/待公布”

## 8. 关键指标
- 首屏渲染成功率 > 99%
- 请求平均耗时 < 2s（缓存命中更快）
- UI 元素不重叠，所有尺寸可读性达标
