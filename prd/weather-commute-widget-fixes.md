# 天气通勤舒适度小组件 UI 优化需求 (Weather Commute Widget UI Fixes)

## 需求背景
目前在 iOS 系统上的中号 (Medium) 和大号 (Large) 天气通勤小组件布局和数据展示存在一些视觉和信息不同步的问题，需要进行统一调整优化。

## 目标与范围
优化 `weather-commute.yaml` / `modules/weather-commute.js` 在中号和大号卡片下的排版对齐，并补全缺失的比对数据。

## 详细执行计划 (Execution Plan)

### 1. 温度柱状图的时间轴对齐优化 (中号/大号)
- **现状**: 逐小时预报列 `hourlyStrip` 中的时间文本（如 10:00, 11:00）因为底部对齐以及每列柱子高度不一致，导致时间文字高低错乱，不美观。
- **修改方案**:
  - 在生成单个时间列 `vstack` 时，计算 `emptyHeight = 20 - barHeight` (基于最大柱高预留空白)。
  - 取消原来基于底部挤压的对齐方式，通过注入动态空白 (`sp(emptyHeight)`) 使底部的温度条和文字在视觉上齐平的同时，将顶部的时间文字推至完全水平对齐。

### 2. 昨日温差对比数据补全 (中号/大号)
- **现状**: 中号和大号的顶部信息区仅包含“舒适度”和“穿衣”标签，缺少“较昨日 (yesterdayDiff)”的温差数据，小号组件中其实已经展示。
- **修改方案**:
  - **中号组件 (`buildMedium`)**: 在 `舒适度` 和 `穿衣` 标签之间，插入 `tag(view.yesterdayDiff.text, ...)`，保持标签形式的一致性。
  - **大号组件 (`buildLarge`)**: 在左上角天气、温度状态下方新增强调横排区域 (`hstack`)，提供 `舒适度` 和 `较昨日` 标签展示。

### 3. 大号组件顶部基础信息对齐优化 (大号)
- **现状**: 大号组件顶部的 天气状况(阴) + 温度范围(最高 14° / 最低 8°) 所在的 `vstack` 默认居中对齐，与右边巨大化的当前温度在视觉排版上显得随意。
- **修改方案**:
  - 明确要求目标 `vstack` 使用 `alignItems: "start"` 属性，使左侧文字块“靠左顶齐”排列，提升卡片秩序感。

## 验证方式
在通过 Surge/Egern 刷新预览该脚本（传入对应的 widgetFamily 为 `systemMedium` 和 `systemLarge`）时，观察卡片排版是否符合预期要求。
